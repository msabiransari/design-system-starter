# Fixture: Angular 7 + Legacy PrimeNG

This fixture validates the ACME design token system against a real Angular 7 application using an older PrimeNG version (matching a production legacy app).

## Status

Implemented as a real Angular 7 + PrimeNG 7 fixture. Build it with Node 12, or use the Docker command from the repository root.

## Purpose

- Prove `base-scoped.css` works in an Angular 7 + PrimeNG hybrid app.
- Validate the `primeng-adapter.css` against the exact PrimeNG version used in production.
- Demonstrate incremental migration: load tokens → adapter → app shim without a full rewrite.

## Node Compatibility

Angular 7 requires Node 10.9–12.x. Do not build this fixture with the repo's normal Node 20+ toolchain.

- `.nvmrc` pins local legacy development to Node 12.
- `Dockerfile` and the root `build:angular7:docker` script build the fixture in `node:12-buster`.
- CI uses a dedicated Docker-backed job, then runs Playwright from the normal root toolchain.

## Local Build

```bash
cd fixtures/angular7-primeng
npm install
npm run build
npm run serve
```

If your workstation is not on Node 12, use Docker from the repository root:

```bash
npm run build:angular7:docker
npm run test:e2e:angular7
```

## Adapter Notes

PrimeNG 7 uses class names like `.ui-button`, `.ui-inputtext`, `.ui-card`, `.ui-table`.
The fixture loads `adapters/primeng7-adapter.css`, which targets those `ui-*` classes.
The modern `adapters/primeng-adapter.css` remains focused on `.p-*` classes for modern PrimeNG.

See `adapters/MATRIX.md` for version coverage.
